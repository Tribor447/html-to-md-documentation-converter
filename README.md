# docs2md

> [!NOTE]
> Актуальная версия краулера находится в директории universal

`docs2md` — краулер документации, который получает HTML-страницы сайта, выделяет основной контент и дерево навигации, преобразует страницы в Markdown и собирает результат в единый файл.

Проект поддерживает как обычные статические сайты, так и документацию, содержимое которой создаётся JavaScript. Для браузерного рендеринга можно опционально установить Playwright.

## Основные возможности

- загрузка одного сайта или списка сайтов из текстового файла;
- параллельный обход страниц с ограничением частоты запросов;
- автоматическое определение движка документации;
- универсальный режим `generic` для сайтов без известного движка;
- сохранение вложенной иерархии навигации;
- преобразование таблиц, блоков кода, предупреждений и других HTML-конструкций;
- скачивание изображений и файлов;
- фильтрация документации по языку и версии;
- HTTP-кеш;
- опциональный рендеринг страниц через Chromium;
- создание `combined.md`, `SUMMARY.md`, JSON/YAML-навигации.

## Требования

- Python 3.9 или новее;
- доступ к сайтам документации;
- Chromium требуется только для браузерного режима Playwright.

## Установка

### 1. Создание виртуального окружения

Linux, macOS или WSL:

```bash
python -m venv .venv
source .venv/bin/activate
```

### 2. Установка проекта

```bash
python -m pip install --upgrade pip
pip install -e .
```

После установки появляется консольная команда:

```bash
docs2md --help
```

Проект также можно запускать как Python-модуль:

```bash
python -m docs2md --help
```

### 3. Установка Playwright

Playwright не обязателен. Он нужен только для страниц, на которых основной HTML появляется после выполнения JavaScript.

```bash
pip install -e '.[browser]'
playwright install chromium
```

## Быстрый старт

### Конвертация одного сайта

```bash
docs2md https://example.com/docs/
```

### Указание папки результата

```bash
docs2md https://example.com/docs/ --out-dir ./result
```

### Несколько сайтов в одной команде

```bash
docs2md \
  https://example.com/docs/ \
  https://another.example.org/manual/
```

### Загрузка ссылок из файла

Создайте `links.txt`:

```text
https://example.com/docs/
https://another.example.org/manual/
#строки, начинающиеся с #, игнорируются
```

Запуск:

```bash
docs2md --targets-file links.txt
```

## Параметры командной строки

| Параметр | Значение по умолчанию | Назначение |
|---|---:|---|
| `urls` | — | Один или несколько начальных URL, переданных позиционно. |
| `--targets-file PATH` | — | Текстовый файл с одним URL на строку. |
| `--out-dir PATH` | `./converted_docs` | Общая корневая папка результатов. Для каждого сайта внутри создаётся отдельный каталог. |
| `--delay SECONDS` | `0.05` | Минимальный интервал между запросами к одному хосту. |
| `--concurrency N` | `12` | Количество параллельных задач загрузки HTML-страниц. |
| `--max-pages N` | без ограничения | Максимальное число сохранённых страниц для одного сайта. |
| `--max-depth N` | `20` | Максимальная глубина переходов от начального URL. |
| `--save-html` | выключено | Сохранять исходные HTML-страницы для отладки. |
| `--use-sitemap` | выключено | Использовать `sitemap.xml` и sitemap-индексы как дополнительные источники URL. |
| `--site-prefix PATH` | определяется автоматически | Жёстко ограничить обход заданным путём, например `/docs/`. |
| `--combined-filename NAME` | `combined.md` | Имя итогового объединённого Markdown-файла. |
| `--browser auto\|always\|never` | `auto` | Управление Playwright-рендерингом. |
| `--browser-timeout-ms N` | `45000` | Тайм-аут загрузки страницы в Chromium. |
| `--admonition-style callout\|html` | `callout` | Формат заметок, предупреждений и подсказок. |
| `--front-matter` | выключено | Добавить YAML front matter в начало объединённого файла. |
| `--profile NAME` | определяется автоматически | Принудительно выбрать профиль движка документации. |
| `--follow-content-links` | выключено | Переходить также по ссылкам внутри основного содержимого страницы. |
| `--no-robots` | выключено | Игнорировать правила `robots.txt`. |
| `--max-asset-mb N` | `25` | Максимальный размер одного скачиваемого файла в мегабайтах. |
| `--asset-concurrency N` | `8` | Число параллельных загрузок изображений и файлов. |
| `--no-assets` | выключено | Не скачивать assets, оставлять абсолютные URL. |
| `--download-external-assets` | выключено | Разрешить загрузку assets с внешних доменов. |
| `--loose-version-scope` | выключено | Разрешить переходы в соседние версии документации. |
| `--language CODE` | `en` | Предпочитаемый язык. Значение `any` отключает языковую фильтрацию. |
| `--cache-dir PATH` | `<out-dir>/.cache` | Отдельный путь для HTTP-кеша. |
| `--no-cache` | выключено | Полностью отключить чтение и запись HTTP-кеша. |

Актуальный список параметров всегда доступен через:

```bash
docs2md --help
```

## Поддерживаемые профили документации

Профиль влияет на CSS-селекторы контента, поиск боковой навигации, фильтрацию служебных страниц и отдельные правила обхода.

| Профиль |
|---|
| `generic` | 
| `docusaurus` | 
| `vitepress` | 
| `sphinx` | 
| `mkdocs` | 
| `maven` | 
| `antora` | 
| `asciidoctor` |
| `javadoc` | 
| `book` | 
| `gitbook` | 

Автоопределение выполняется по структуре DOM, классам, метатегам, URL и характерным элементам страницы.

## Как строится навигация

Краулер не ограничивается одним типом меню. Он пытается восстановить дерево несколькими способами:

1. выбирает наиболее вероятный контейнер боковой навигации;
2. разбирает вложенные `ul`/`ol` и сохраняет их глубину;
3. обрабатывает заголовки категорий без ссылок;
4. извлекает меню с разных страниц;
5. оценивает качество полученных деревьев;
6. объединяет деревья без дублирования узлов;
7. удаляет ссылки на страницы, которые не вошли в результат;
8. при отсутствии нормального меню строит дерево по заголовкам страницы или URL-путям.

## Ограничение области обхода

Краулер старается не переходить из документации в блог, новости, страницы загрузок, отчёты сборки, архивы релизов и соседние проекты.

Область определяется по:

- начальному URL;
- автоматически найденному `site_prefix`;
- версии в пути или query-параметрах;
- профилю сайта;
- языку страницы;
- правилам исключения URL;
- `robots.txt`.

Когда автоматический префикс выбран неверно, задайте его явно:

```bash
docs2md https://example.com/product/docs/start --site-prefix /product/docs/
```

По умолчанию краулер остаётся в версии, указанной начальным URL. Для обхода соседних версий добавьте:

```bash
--loose-version-scope
```

## Работа с языками

По умолчанию используется:

```bash
--language en
```

Краулер анализирует языковые сегменты URL, query-параметры, `hreflang`, атрибут `html[lang]`.

Другой язык:

```bash
docs2md https://example.com/docs/ru/ --language ru
```

Отключение языковой фильтрации:

```bash
docs2md https://example.com/docs/ --language any
```

## Assets: изображения и файлы

По умолчанию краулер скачивает assets только с того же домена, что и документация.

Скачиваются, например:

- изображения;
- PDF;
- архивы;
- текстовые и конфигурационные файлы;
- файлы, на которые ссылается основной контент.

Внешние assets остаются абсолютными ссылками. Чтобы разрешить их загрузку:

```bash
--download-external-assets
```

Чтобы не скачивать ничего:

```bash
--no-assets
```

Ограничение размера одного файла:

```bash
--max-asset-mb 10
```

## HTTP-кеш

Кеш уменьшает число повторных запросов и ускоряет перезапуски.

По умолчанию он находится здесь:

```text
<out-dir>/.cache/
```

Другой каталог:

```bash
--cache-dir ./crawler-cache
```

Полное отключение:

```bash
--no-cache
```

## Формат результата

Для каждого начального сайта создаётся отдельная папка внутри `--out-dir`.

### `combined.md`

Главный результат. Содержит:

1. необязательный YAML front matter;
2. заголовок сайта;
3. иерархическую навигацию;
4. содержимое всех страниц;
5. локальные anchors;

Внутренние ссылки между загруженными страницами преобразуются в ссылки на anchors внутри этого файла.

### `SUMMARY.md`

Краткое дерево навигации в Markdown.

### `navigation.json`

json дерева навигации. Каждый узел может содержать:

- `title`;
- `kind`;
- `source_url`;
- `anchor`;
- `virtual_path`;
- `children`.

### `navigation.yml`

То же дерево навигации в YAML-формате.

### `site_manifest.json`

Общая информация о результате:

- начальный URL;
- префикс сайта;
- профиль;
- число страниц;
- число assets;
- список страниц;
- заголовки;
- anchors;
- виртуальные пути;
- глубина обхода;
- оценка качества контента.

### `assets/`

Локальные копии скачанных изображений и файлов. 

### `html_cache/`

Исходные HTML-страницы, создаваемые только при использовании `--save-html`. Этот каталог предназначен для анализа неправильного выбора контента или навигации.

## Название выходной папки

Общая корневая папка задаётся параметром:

```bash
--out-dir ./result
```

Внутренняя папка конкретного сайта создаётся автоматически из домена и `site_prefix`. 

Например:

```text
https://example.com/docs/
```

может получить каталог:

```text
result/example_docs/
```
